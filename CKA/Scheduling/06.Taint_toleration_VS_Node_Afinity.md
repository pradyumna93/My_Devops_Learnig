
# Taints & Tolerations vs Node Affinity

## 1. Core Difference (EXAM GOLD ⭐)

| Feature              | Taints & Tolerations                  | Node Affinity                     |
| -------------------- | ------------------------------------- | --------------------------------- |
| Applied on           | Nodes (taint) + Pods (toleration)     | Pods                              |
| Purpose              | Control **which pods a node accepts** | Control **where pods are placed** |
| Push or Pull         | Node **pushes pods away**             | Pod **pulls itself to node**      |
| Guarantees placement | ❌ No                                  | ✅ Yes (with Required)             |
| Eviction support     | ✅ (NoExecute)                         | ❌                                 |
| Exam importance      | High                                  | Very High                         |

---

## 2. When to Use What (Very Important)

### Use **Taints & Tolerations** when:

* You want to **protect a node**
* Example:

  * Master nodes
  * Dedicated nodes
  * GPU nodes

### Use **Node Affinity** when:

* You want to **control pod placement**
* Example:

  * High-resource pods
  * Region / zone based placement

### Best Practice:

👉 **Use both together** for full control

---

## 3. Real-World Example

Requirement:

* Only data-processing pods should run on large nodes
* Other pods must not enter

Solution:

1. Taint large node
2. Add toleration to data-processing pods
3. Add node affinity to force placement

# Hands-On Labs (CKA Practical)

## Lab 1: Dedicated High-Resource Node

### Step 1: Label Node

```bash
kubectl label nodes node1 size=large
```

### Step 2: Taint Node

```bash
kubectl taint nodes node1 workload=high:NoSchedule
```

---

## Lab 2: Pod That Runs Only on Large Node

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: heavy-pod
spec:
  containers:
  - name: app
    image: nginx
  tolerations:
  - key: "workload"
    operator: "Equal"
    value: "high"
    effect: "NoSchedule"
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - key: size
            operator: In
            values:
            - large
```

✅ Result:

* Pod runs **only on node1**

---

## Lab 3: Soft Placement (Preferred)

Change:

```yaml
preferredDuringSchedulingIgnoredDuringExecution:
```

➡️ Pod prefers large node but can run elsewhere

---

## Lab 4: Test Eviction

```bash
kubectl taint nodes node1 test=true:NoExecute
```

Observe:

* Pods without toleration are evicted

---

# Quick Memory Tricks 🧠

* **Taints = Node rules**
* **Affinity = Pod rules**
* **NoExecute = Eviction**
* **Required = Strict**
* **Preferred = Best effort**
* **IgnoredDuringExecution = No eviction**
