### **1. How many namespaces exist in the system? Get using command.**

**Answer:**

To list all namespaces in the cluster:

```bash
kubectl get namespaces
```

or

```bash
kubectl get ns
```

âœ… This command will display all namespaces such as:

* default
* kube-system
* kube-public
* kube-node-lease
* (and any custom ones like dev, prod, etc.)

### **2. Get how many pods are present in the `dev` namespace (all possible commands).**

**Answer:**

You can use any of the following commands:

```bash
kubectl get pods -n dev
```

or

```bash
kubectl get po --namespace=dev
```

To get a count only:

```bash
kubectl get pods -n dev --no-headers | wc -l
```

To view detailed information about pods:

```bash
kubectl describe pods -n dev
```

To view pods in all namespaces (and check where `dev` pods are):

```bash
kubectl get pods --all-namespaces
```

### **3. Create a pod in the `dev` namespace using the below attributes.**

**Attributes:**

* **Pod name:** redis
* **Image:** redis

**Answer:**

#### **Option 1: Using Command**

```bash
kubectl run redis --image=redis -n dev
```

#### **Option 2: Using YAML file**

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: redis
  namespace: dev
spec:
  containers:
    - name: redis
      image: redis
```

Then apply:

```bash
kubectl apply -f redis-pod.yaml
```

### **4. Which namespace has a pod named `blue` in it?**

**Answer:**

You can check using the **all namespaces** flag:

```bash
kubectl get pods --all-namespaces | grep blue
```

âœ… Output Example:

```
dev          blue          1/1     Running   0     2m
```

Here, the **`dev` namespace** has the `blue` pod.
