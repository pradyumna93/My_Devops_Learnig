# 🌐 Kubernetes Nodes

This note explains the concept of Kubernetes nodes, their types, and how to inspect them in a cluster.

---

## 1. What is a Node?

A node is either a physical machine or a virtual machine that runs your applications in a Kubernetes cluster.

Once your application is packaged as a container, Kubernetes runs it on one of the available nodes.

---

## 2. Types of Nodes

Kubernetes nodes are broadly divided into two categories:

### 1) Control Plane Node

The control plane node manages the Kubernetes cluster.

It runs core cluster components such as:

- API Server
- Scheduler
- Controller Manager
- etcd

This node is responsible for making cluster-level decisions.

### 2) Worker Node

Worker nodes are used to run the actual application workloads.

They host:

- pods
- container runtime
- kubelet
- kube-proxy

---

## 3. Node Responsibilities

### Control Plane Node

- manages the cluster
- schedules workloads
- monitors cluster state
- stores cluster configuration data

### Worker Node

- runs pods
- executes container workloads
- reports node health to the control plane

---

## 4. List Nodes in a Cluster

To list all nodes, use:

```bash
kubectl get nodes
```

Example output may show:

- one control plane node
- one or more worker nodes

---

## 5. Get Detailed Node Information

To get more details about nodes, use:

```bash
kubectl get nodes -o wide
```

This shows details such as:

- internal IP addresses
- OS image
- container runtime version
- node status

---

## 6. Describe a Node

To inspect a node in more detail:

```bash
kubectl describe node <node-name>
```

This gives information such as:

- node roles
- capacity and allocatable resources
- conditions
- addresses
- system information

---

## 7. Key Takeaway

In Kubernetes:

- control plane nodes manage the cluster
- worker nodes run the actual application workloads

---

## 8. Quick Revision Summary

> Nodes are the machines that make up a Kubernetes cluster. Control plane nodes manage the cluster, while worker nodes run the application workloads.
