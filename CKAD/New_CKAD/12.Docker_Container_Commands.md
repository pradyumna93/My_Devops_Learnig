# 🐳 Docker Container Commands

This note summarizes the basic container commands you should know for the CCAD/CKAD exam when working with Docker images and containers.

---

## 1. Run a Container in the Foreground

To create and start a container from an image, use:

```bash
docker container run nginx:latest
```

### What happens

- Docker checks whether the image exists locally
- if not, it pulls it from the default registry (Docker Hub)
- it creates a container from that image
- the container runs in the foreground

If you want to give the container a custom name:

```bash
docker container run --name my-nginx nginx:latest
```

---

## 2. Run a Container in Detached Mode

If you want the container to run in the background, use the `-d` flag:

```bash
docker container run -d --name nginx-background nginx:latest
```

This starts the container and returns the container ID.

---

## 3. List Running Containers

To list containers that are currently running:

```bash
docker ps
```

To list all containers, including stopped ones:

```bash
docker ps -a
```

---

## 4. Stop a Container

To stop a running container:

```bash
docker container stop <container_name_or_id>
```

Example:

```bash
docker container stop nginx-background
```

---

## 5. Start a Stopped Container

To start a container that has already been created but is stopped:

```bash
docker container start <container_name_or_id>
```

---

## 6. Remove a Container

To delete a container:

```bash
docker container rm <container_name_or_id>
```

> A container must usually be stopped before it can be removed.

---

## 7. View Container Logs

To inspect what a container printed or logged:

```bash
docker container logs <container_name_or_id>
```

This is very useful for troubleshooting.

---

## 8. Execute a Command Inside a Running Container

To open a shell inside a running container:

```bash
docker container exec -it <container_name_or_id> sh
```

This is helpful when you want to inspect files or troubleshoot the container environment.

---

## 9. Inspect Container Details

To get full details about a container in JSON format:

```bash
docker container inspect <container_name_or_id>
```

This gives useful information such as:

- network settings
- environment variables
- mounted volumes
- configuration details

---

## 10. Login to a Registry

To push images to Docker Hub, you first need to log in:

```bash
docker login
```

You will be prompted for your username and password.

---

## 11. Tag an Image for a Registry

Before pushing an image to a registry, tag it properly:

```bash
docker image tag app:1.0 username/app:1.0
```

This helps Docker know where to push the image.

---

## 12. Push an Image to a Registry

To upload an image to a registry:

```bash
docker push username/app:1.0
```

---

## 13. Pull an Image from a Registry

To download an image from a registry:

```bash
docker pull username/app:1.0
```

---

## 14. Example Workflow

A simple workflow for working with containers is:

1. Build an image
2. Run a container from that image
3. Check logs or inspect the container if needed
4. Stop or remove the container when done
5. Tag and push the image to a registry if needed

Example:

```bash
docker build -t app:1.0 .
docker run --name app-demo app:1.0
docker ps
docker logs app-demo
docker stop app-demo
docker rm app-demo
```

---

## 15. Key Takeaways

For exam preparation, focus on these core commands:

- `docker run`
- `docker ps`
- `docker stop`
- `docker start`
- `docker rm`
- `docker logs`
- `docker exec`
- `docker inspect`
- `docker pull`
- `docker push`

---

## 16. Quick Revision Summary

> Docker containers are created from images using `docker run`, and basic lifecycle commands like `ps`, `stop`, `start`, `rm`, `logs`, and `inspect` are enough for most CCAD/CKAD exam tasks.
