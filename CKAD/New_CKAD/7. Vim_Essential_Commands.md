# Vim Editor Essential Commands for Kubernetes Exams

## Introduction

While you can use various editors to work with manifest files, Vim is highly recommended for Kubernetes exams. Vim provides several advantages:

- **Fast editing** - Keyboard-driven commands save time
- **YAML-ready** - Excellent for YAML file manipulation
- **Terminal-native** - Runs efficiently in terminal environments
- **Powerful tools** - Built-in search, replace, copy, and delete functionality
- **Exam-optimized** - Pre-configured with YAML-friendly settings

Learning essential Vim commands can save you several minutes per task during the exam, which adds up when you have multiple tasks to complete.

---

## Typical Vim Workflow for Kubernetes Tasks

```
1. Generate YAML manifest using kubectl dry-run command
   ↓
2. Open the YAML file in Vim editor
   ↓
3. Edit and customize the manifest according to task requirements
   ↓
4. Save and exit the file
   ↓
5. Apply the manifest using kubectl apply command
```

## Opening and Closing Files

### Opening a File

**Syntax:**
```bash
vim <filename>
```

**Example:**
```bash
vim nginx-pod.yaml
```

Once opened, you're in **read-only mode** by default. Changes won't be saved unless you explicitly make them in insert mode and save.

### Saving and Exiting

#### Save and Exit

```
:wq
```

**Steps:**
1. Press `:` (colon) to enter command mode
2. Type `w` (write)
3. Type `q` (quit)
4. Press Enter

**Result:** Changes are saved and you exit Vim

#### Exit Without Saving

```
:q!
```

**Steps:**
1. Press `:` (colon) to enter command mode
2. Type `q` (quit)
3. Type `!` (force)
4. Press Enter

**Result:** All changes are discarded and you exit Vim

### Quick Reference: File Operations

| Command | Action |
|---------|--------|
| `vim filename` | Open a file |
| `:wq` | Save and exit |
| `:q!` | Exit without saving |
| `:w` | Save without exiting |
| `:q` | Exit (fails if unsaved changes) |

## Navigation Commands

### Jump to First Line

```
gg
```

Moves cursor to the first line and first character of the file.

### Jump to Last Line

```
G
```

(Capital G) Moves cursor to the last line of the file.

### Move to Next Word

```
w
```

Moves cursor to the beginning of the next word. Pressing `w` repeatedly moves through words one by one.

**Example Navigation:**
```yaml
# File content
kind: Pod
metadata:
  name: nginx-pod

# Cursor movements with 'w':
kind → Pod → metadata → name → nginx-pod
```

### Quick Reference: Navigation

| Command | Action |
|---------|--------|
| `gg` | Go to first line |
| `G` | Go to last line |
| `w` | Move to next word |
| `↑ ↓ ← →` | Arrow keys for single character movement |

## Editing Modes and Commands

### Enter Insert Mode

**Command:** `i`

Pressing `i` enters **Insert Mode**, which allows you to type and edit text. You'll see "insert" displayed at the bottom left.

**Example:**
```
Escape key pressed (read-only mode)
  ↓
Press 'i'
  ↓
Insert mode activated (now can type)
```

### Exit Insert Mode

**Command:** `Escape`

Pressing `Escape` returns you from Insert Mode to **Normal (read-only) Mode**.

### Add New Line and Enter Insert Mode

**Command:** `o`

Pressing `o` in Normal Mode:
1. Creates a new line below the current line
2. Automatically enters Insert Mode
3. Cursor is ready for typing

**Shorthand:** Instead of `Escape` → move → `i`, just use `o`

### Quick Reference: Modes

| Command | Action |
|---------|--------|
| `i` | Enter Insert Mode at cursor |
| `Escape` | Exit Insert Mode to Normal Mode |
| `o` | Create new line below and enter Insert Mode |
| `O` | Create new line above and enter Insert Mode |

## Copy and Paste

### Copy a Single Line

**Command:** `yy`

"yy" stands for "yank yank" (yank = copy in Vim terminology).

**Steps:**
1. Position cursor on the line you want to copy
2. Press `yy` (no visual feedback, but line is copied)
3. Navigate to target location
4. Press `p` to paste

**Example:**
```yaml
Original file:
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:

Position on line 3, press 'yy'
Navigate to line 6, press 'p'

Result:
kind: Pod
metadata:
  name: nginx-pod
  name: nginx-pod        ← pasted line
spec:
  containers:
```

### Copy Multiple Lines

**Command:** `<number>yy`

Prefix the number of lines you want to copy.

**Example: Copy 3 lines**
```
3yy
```

This copies the current line and the next 2 lines (3 lines total).

**Steps:**
1. Position cursor on first line to copy
2. Type `3yy` (means: copy 3 lines)
3. Bottom left shows "3 lines yanked"
4. Navigate to target location
5. Press `p` to paste all 3 lines

### Paste

**Command:** `p`

Pastes the copied content below the current line.

**Note:** Content is pasted starting from the line after the cursor position.

### Quick Reference: Copy/Paste

| Command | Action |
|---------|--------|
| `yy` | Copy current line |
| `3yy` | Copy 3 lines (starting from current) |
| `5yy` | Copy 5 lines |
| `p` | Paste below cursor |
| `P` | Paste above cursor |

## Delete Operations

### Delete a Single Line

**Command:** `dd`

"dd" deletes the entire line where the cursor is positioned.

**Steps:**
1. Position cursor on line to delete
2. Press `dd`
3. Line is deleted and lines below shift up

### Delete Multiple Lines

**Command:** `<number>dd`

Prefix the number of lines you want to delete.

**Example: Delete 3 lines**
```
3dd
```

This deletes the current line and the next 2 lines.

**Practical Example:**
```yaml
Before (cursor on line 3):
1: kind: Pod
2: metadata:
3: name: nginx-pod
4: labels:
5:   app: web

Execute: 3dd

After:
1: kind: Pod
2: metadata:
3: labels:
4:   app: web
```

### Quick Reference: Delete

| Command | Action |
|---------|--------|
| `dd` | Delete current line |
| `3dd` | Delete 3 lines (current + next 2) |
| `5dd` | Delete 5 lines |
| `x` | Delete single character |
| `dw` | Delete word |

## Undo and Redo

### Undo Last Change

**Command:** `u`

Pressing `u` undoes the most recent change. Pressing `u` repeatedly undoes previous changes one by one.

**Example:**
```
Press u once      → Undo last change
Press u again     → Undo change before that
Press u again     → Undo change before that
... (keep pressing u to go back further)
```

### Redo

**Command:** `Ctrl + r`

Pressing `Ctrl + r` redoes the last undone change. Works opposite to undo.

**Example Workflow:**
```
1. Delete line (dd)
2. Delete another line (dd)
3. Undo last deletion (u)      → Line 2 reappears
4. Undo before that (u)        → Line 1 reappears
5. Redo last undo (Ctrl + r)   → Line 1 disappears again
6. Redo again (Ctrl + r)       → Line 2 disappears again
```

### Quick Reference: Undo/Redo

| Command | Action |
|---------|--------|
| `u` | Undo last change |
| `Ctrl + r` | Redo last undone change |

## YAML-Specific Operations: Indentation

### Important: YAML Spacing

YAML is **very sensitive to spacing and indentation**. Even one extra or missing space can break your manifest. The pre-configured Vim in exam terminals handles YAML spacing automatically, but you need to know how to indent correctly.

### Indent to the Right

**Command:** `>>`

Pressing `>>` indents the current line to the right.

**Default:** Usually 2 spaces per indent level (pre-configured for YAML)

**Steps:**
1. Position cursor on line to indent
2. Press `>>`
3. Line moves right by 2 spaces

### Indent Multiple Times

**Command:** `<number>>>`

Prefix with the number of times to indent.

**Example:**
```
2>>    (indent 2 times = 4 spaces right)
3>>    (indent 3 times = 6 spaces right)
```

### Indent Multiple Lines

**Command:** `<number>` `>>`

Select multiple lines and indent them all at once.

**Steps:**
1. Position cursor on first line to indent
2. Type `2>>` (indent current + next line, 2 times each)
3. Both lines move right together

**Example:**
```yaml
Before:
kind: Pod
spec:

Position on 'kind', press 2>>:

After:
    kind: Pod
    spec:
```

### Unindent to the Left

**Command:** `<<`

Pressing `<<` unindents the current line to the left.

### Unindent Multiple Lines

**Command:** `<number><<`

**Example:**
```
2<<    (unindent 2 times = 4 spaces left)
3<<    (unindent 3 times = 6 spaces left)
```

### Quick Reference: Indentation

| Command | Action |
|---------|--------|
| `>>` | Indent line right (2 spaces) |
| `2>>` | Indent line right twice (4 spaces) |
| `<<` | Unindent line left (2 spaces) |
| `2<<` | Unindent line left twice (4 spaces) |
| `2>>` on multiple lines | Indent 2 lines right |

## Visual Mode: Select Multiple Lines

### Enter Visual Line Mode

**Command:** `Shift + v`

Pressing `Shift + v` enters **Visual Line Mode**, allowing you to select multiple lines at once.

**Indication:** Bottom left shows "visual line"

### Select Lines in Visual Mode

Once in Visual Line Mode:
- Press `↑` (up arrow) to select lines upward
- Press `↓` (down arrow) to select lines downward

### Perform Operations on Selected Lines

After selecting multiple lines, you can:
- **Indent:** Press `>>` or `2>>` to indent all selected lines
- **Unindent:** Press `<<` or `2<<` to unindent all selected lines
- **Delete:** Press `dd` to delete all selected lines
- **Copy:** Press `yy` to copy all selected lines
- **Paste:** Press `p` to paste after selection

### Visual Mode Example

**Task:** Indent lines 3-5 in a YAML file

```yaml
1: kind: Pod
2: metadata:
3:   name: nginx-pod
4:   namespace: default
5:   labels:
6:     app: web

Steps:
1. Position cursor on line 3
2. Press Shift + v (enter visual line mode)
3. Press down arrow twice (select lines 3, 4, 5)
4. Press >> or 2>> to indent selected lines

Result:
1: kind: Pod
2: metadata:
3:     name: nginx-pod
4:     namespace: default
5:     labels:
6:     app: web
```

### Quick Reference: Visual Mode

| Command | Action |
|---------|--------|
| `Shift + v` | Enter Visual Line Mode |
| `↓` | Select next line |
| `↑` | Select previous line |
| `>>` | Indent selected lines |
| `<<` | Unindent selected lines |
| `dd` | Delete selected lines |
| `yy` | Copy selected lines |

## Search Operations

### Search for Text

**Command:** `/` (forward slash)

Pressing `/` opens the search prompt at the bottom of the screen.

**Steps:**
1. Press `/`
2. Type the search term (e.g., "nginx")
3. Press Enter
4. Cursor jumps to first occurrence

### Find Next Occurrence

**Command:** `n`

Pressing `n` (lowercase) moves to the next occurrence of the search term.

### Find Previous Occurrence

**Command:** `N`

Pressing `N` (uppercase) moves to the previous occurrence of the search term.

### Search Example

```yaml
File content:
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
  - name: nginx-container
    image: nginx:latest

Search workflow:
1. Press /
2. Type: nginx
3. Press Enter → Jumps to first "nginx" (line 3)
4. Press n → Jumps to next "nginx" (line 7)
5. Press n → Jumps to next "nginx" (line 8)
6. Press N → Jumps back to previous "nginx" (line 7)
```

### Important: Search Starts from Cursor

**Note:** Search starts from the cursor position and moves forward. If you want to search from the beginning:
1. Press `gg` to go to the start
2. Press `/` and enter search term
3. Press Enter

### Quick Reference: Search

| Command | Action |
|---------|--------|
| `/` | Open search prompt |
| `n` | Find next occurrence |
| `N` | Find previous occurrence |
| `gg` then `/` | Search from beginning of file |

## Pre-Configured Vim Settings in Exam Terminal

Your exam terminal comes with Vim pre-configured for YAML editing. These settings are already applied:

### Important Pre-Configured Settings

| Setting | Value | Meaning |
|---------|-------|---------|
| `expandtab` | On | Tabs are expanded to spaces |
| `tabstop` | 2 | Each tab represents 2 spaces |
| `shiftwidth` | 2 | Indent/unindent moves by 2 spaces |

### What This Means for You

- **Don't worry about tab/space confusion** - Tabs automatically convert to spaces
- **Indentation is consistent** - Every indent level is 2 spaces (perfect for YAML)
- **YAML-friendly by default** - No need to manually configure
- **Exam-optimized** - Settings are already tuned for Kubernetes tasks

## Practical Exam Scenario

**Task:** Edit a deployment YAML file

```
1. Generate YAML:
   kubectl create deployment my-app --image=nginx --dry-run=client -o yaml > deployment.yaml

2. Open in Vim:
   vim deployment.yaml

3. Navigate to the section you want to edit:
   /replicas (search for replicas)
   n (find next occurrence)

4. Edit the value:
   i (enter insert mode)
   Change value
   Escape (exit insert mode)

5. Add a new line for an environment variable:
   o (new line + insert mode)
   Type the new content
   Escape (exit insert mode)

6. Indent if needed:
   >> (indent right)
   << (indent left)

7. Save and exit:
   :wq

8. Apply the changes:
   kubectl apply -f deployment.yaml
```

## Practice Tips

### Before the Exam

1. **Memorize the essential commands** - Don't look them up during the exam
2. **Practice in your terminal** - Get comfortable with Vim before exam day
3. **Learn the workflow** - Generate → Edit → Apply
4. **Practice copy/paste** - It's one of the most-used operations
5. **Understand YAML indentation** - Spacing is critical
6. **Use undo liberally** - Don't be afraid to experiment; `u` is your friend

### During the Exam

1. **Avoid learning new commands** - Stick with the ones you practiced
2. **Use search (`/`) liberally** - It's faster than scrolling
3. **Take advantage of `yy` + `p`** - Faster than manually retyping similar sections
4. **Use Visual Mode (`Shift + v`)** - When you need to indent multiple lines
5. **Remember `:wq`** - Always save before applying with kubectl
