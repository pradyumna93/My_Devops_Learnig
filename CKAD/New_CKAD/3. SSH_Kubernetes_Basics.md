# SSH Commands for Kubernetes Exams

## Introduction

SSH (Secure Shell) is a network protocol that allows you to connect to computers over an unsecured network. In Kubernetes exams, you'll need to connect to pre-configured clusters that have been provisioned as part of the exam setup. The good news is that the SSH commands required are straightforward and easy to master.

## Why SSH Matters for Exam Preparation

Understanding SSH is crucial for several reasons:

1. **Connect to Remote Nodes** - You need to connect to remote Kubernetes nodes that have been provisioned as part of the exam tasks
2. **Lab Environment Access** - Online lab environments often require SSH connections for exam preparation
3. **Practice in Cloud Terminals** - Whether using terminal, CLI, or cloud options, SSH commands help you practice in cloud-based environments
4. **Node Switching** - When you have multiple nodes (cluster nodes, worker nodes), you need to switch between them, execute commands, and exit

## Basic SSH Commands

### Command 1: Connect to a Node

**Syntax:**
```bash
ssh <hostname>
# or
ssh <IP_address>
```

**Example:**
```bash
ssh kubernetes0012
```

**What happens:**
- You'll see a prompt like `Last login: ...`
- You're now logged into the specified node
- The terminal is open and ready for commands

**Execute commands on the node:**
```bash
kubectl get nodes
```

### Command 2: Exit from a Node

**Option 1 - Using exit command:**
```bash
exit
```

**Option 2 - Using keyboard shortcut:**
```
Ctrl + D
```

Both will exit the current node and return you to the previous/parent node.

---

## Advanced Usage: Single Command Execution

If you want to execute a single command without logging in, staying in the terminal, and then exiting, you can combine all steps into one statement:

**Syntax:**
```bash
ssh <hostname> '<command>'
```

**Example:**
```bash
ssh kubernetes0012 'kubectl get nodes'
```

**What happens:**
- SSH connects to the specified node
- Executes the command
- Returns the results
- Automatically disconnects

This is efficient when you only need to run one command and don't need to stay in the terminal.

## Two Approaches Summary

| Approach | Use Case | Steps |
|----------|----------|-------|
| **Interactive** | Multiple commands on a node | 1. SSH to node<br>2. Execute commands<br>3. Exit |
| **Single Command** | One-off commands | 1. SSH + command in one line |

## Quick Reference

| Action | Command |
|--------|---------|
| Connect to a node | `ssh kubernetes0012` |
| Exit from node | `exit` or `Ctrl + D` |
| Run single command | `ssh kubernetes0012 'kubectl get pods'` |
| Connect with IP | `ssh 192.168.1.10` |

---

## Tips for Exam Success

- Practice SSH commands before the exam to build muscle memory
- Familiarize yourself with the node names/IPs provided in your exam setup
- Remember that commands executed via SSH run on the **remote node**, not your local machine
- Use single-command SSH when you just need quick information
- Use interactive SSH when you need to execute multiple commands and explore
