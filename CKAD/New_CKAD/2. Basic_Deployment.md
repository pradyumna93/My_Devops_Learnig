# Kubernetes Deployment Basics

In this lecture, we will learn how to create and manage a Kubernetes deployment using an online environment, so you do not need to set up a cluster on your local machine.

## 1. Getting Started with KillerCoda

1. Open your browser and go to https://killercoda.com/
2. Sign up for an account if you do not already have one.
3. Log in to your account.
4. Go to the **CKAD / Kubernetes Certification** section.
5. Click on the **Playground** option.

This gives you a live Kubernetes sandbox where you can practice commands safely.

## 2. Check Available Nodes

Once the playground is ready, verify that the cluster is working:

```bash
kubectl get nodes
```

You should see at least two nodes, usually:
- `control-plane`
- `node01`

This confirms that your cluster is up and running.

## 3. Create a Deployment

A deployment manages the desired number of replicas for your application.

To create a deployment that runs an NGINX server:

```bash
kubectl create deployment nginx-deploy --image=nginx
```

This command creates a deployment named `nginx-deploy` using the official NGINX image from Docker Hub.

## 4. Check Deployment Status

To see a list of deployments:

```bash
kubectl get deployments
```

To get more details about the deployment:

```bash
kubectl describe deployment nginx-deploy
```

You will see information such as the number of replicas. By default, it starts with 1 running pod.

## 5. Scale the Deployment

You can increase or decrease the number of replicas as needed.

### Scale up to 4 replicas

```bash
kubectl scale deployment nginx-deploy --replicas=4
```

### Scale down to 2 replicas

```bash
kubectl scale deployment nginx-deploy --replicas=2
```

You can verify the current resources using:

```bash
kubectl get all
```

## 6. Delete the Deployment

When you are done practicing, remove the deployment:

```bash
kubectl delete deployment nginx-deploy
```

After deletion, running:

```bash
kubectl get all
```

should show that the deployment and its pods are no longer present.

## Conclusion

This exercise covers the basics of creating, checking, scaling, and deleting a Kubernetes deployment. These are important skills for the Kubernetes exam and for real-world cluster management.
