# 🏷️ Kubernetes Labels

This note explains what labels are, why they are used, and how to apply them to Kubernetes resources.

---

## 1. What Are Labels?

Labels are key-value pairs that you attach to Kubernetes objects.

They are used to:

- identify resources easily
- organize resources logically
- group related objects together
- filter resources using `kubectl`

Example:

```yaml
app: nginx
environment: production
```

---

## 2. Why Are Labels Important?

Labels help in two main ways:

### 1) Querying and Filtering

Once labels are attached, you can select resources based on those labels.

Example:

```bash
kubectl get pods -l environment=production
```

### 2) Grouping Resources

You can group related resources together by assigning the same labels.

Example:

- all production resources can have `environment=production`
- all frontend resources can have `app=frontend`

---

## 3. Common Label Use Cases

### Environment Label

Used to identify the environment of a resource:

- `environment=development`
- `environment=staging`
- `environment=production`

### Application Type Label

Used to identify the type of application:

- `app=frontend`
- `app=backend`
- `app=database`

### Version Label

Used to identify the version of an application:

- `version=v1`
- `version=v2`

### Team Label

Used to group resources by team or ownership:

- `team=devops`
- `team=platform`

---

## 4. Apply a Label to an Existing Resource

To add a label to an existing resource, use:

```bash
kubectl label pod nginx-pod environment=production
```

Example for a Deployment:

```bash
kubectl label deployment webapp app=frontend
```

---

## 5. Apply Labels During Resource Creation

You can also add labels while creating a resource.

Example:

```bash
kubectl run nginx --image=nginx --labels=environment=staging,app=frontend
```

This creates a Pod and assigns the labels immediately.

---

## 6. Apply Labels to Multiple Resources

You can also apply a label to all resources of a specific type in a namespace.

Example:

```bash
kubectl label pods --all environment=production -n staging
```

This applies the label to all Pods in the `staging` namespace.

---

## 7. View Labels

To view labels on a resource:

```bash
kubectl get pods --show-labels
```

To get pods matching a label:

```bash
kubectl get pods -l environment=production
```

---

## 8. Update an Existing Label

To change the value of an existing label, use `--overwrite`.

Example:

```bash
kubectl label pod nginx-pod environment=production --overwrite
```

If you do not use `--overwrite`, Kubernetes will return an error when the label already exists with a different value.

---

## 9. Remove a Label

To remove a label from a resource:

```bash
kubectl label pod nginx-pod environment-
```

This deletes the `environment` label from the Pod.

---

## 10. View Labels on Resources

To see labels on a resource, use:

```bash
kubectl get pods --show-labels
```

This displays the labels attached to the Pods.

You can also inspect labels using:

```bash
kubectl describe pod nginx-pod
```

---

## 11. Key Takeaway

Labels are simple key-value metadata attached to Kubernetes resources. They are very useful for filtering, grouping, and organizing objects in a cluster.

---

## 12. Quick Revision Summary

> Labels are used to identify and organize Kubernetes resources. They help with filtering, grouping, and managing objects more efficiently.
