# CCAD Exam Skills Overview

Let us explore the key skills measured in the CKAD exam. This will give you a clear understanding of the areas you need to be comfortable with when working with cloud-native applications.

## 1. Application Design and Build (About 20%)

This section focuses on designing, building, and modifying container images.

You should know how to:
- Define and build container images
- Choose the right workload resources for the task
- Understand when to use:
  - DaemonSets
  - Jobs
  - CronJobs
  - Deployments
- Work with multi-container pod design patterns
- Use init containers and sidecar containers
- Manage storage using volumes, both persistent and ephemeral

## 2. Application Deployment (About 20%)

This section covers how to deploy applications using Kubernetes primitives.

You should understand:
- Deployment strategies such as blue-green and canary deployments
- Rolling updates for smooth application updates
- How deployments work in Kubernetes
- Tools such as Helm for packaging and deployment
- Kubernetes configuration customization techniques

## 3. Application Observability and Maintenance (About 15%)

This section is about keeping applications healthy and monitoring them effectively.

You should be able to:
- Understand API deprecations and when APIs are no longer supported
- Implement probes and health checks
- Use `kubectl` to monitor applications in real time
- Check container logs to troubleshoot issues
- Apply debugging techniques inside Kubernetes environments

## 4. Application Environment, Configuration, and Security (About 25%)

This is the largest section of the exam.

You should know how to:
- Use Custom Resource Definitions (CRDs) and Operators
- Understand authentication, authorization, and admission control
- Manage configuration with ConfigMaps
- Set resource requests and limits for CPU and memory
- Create and use Secrets for sensitive data
- Work with ServiceAccounts for pod identity
- Use SecurityContext to define privilege and access control settings
- Understand Linux capabilities and permission control
- Set resource quotas to manage cluster usage effectively

## 5. Services and Networking (About 20%)

This section covers communication and access for applications.

You should understand:
- NetworkPolicies for controlling traffic between pods and external systems
- Different service types:
  - ClusterIP
  - NodePort
  - LoadBalancer
- How to expose applications using Ingress rules
- Troubleshooting and managing application access

## Final Note

You can find the exam curriculum breakdown in the CNCF GitHub repository.

Although the percentages are useful, it is best not to focus too much on them. The main goal is to be confident in all the topics. A strong preparation strategy is to study every area carefully instead of worrying too much about exact weighting.
