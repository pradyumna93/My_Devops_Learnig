# 🐳 Docker Image Build and Management Commands

This note summarizes the basic Docker commands you should know for the CCAD/CKAD exam when working with container images.

---

## 1. Verify Docker Installation

Before working with Docker, make sure Docker is installed on your machine.

```bash
docker version
```

This command shows information about:

- the Docker client
- the Docker server/engine
- the API versions supported

### What the output means

- Client version → version of the Docker CLI
- API version → version of the Docker API used by the client
- Server version → version of the Docker Engine
- Server API version → API version supported by the engine

---

## 2. Get Help for Docker Commands

If you are unsure about a Docker command, use:

```bash
docker --help
```

You can also get help for a specific command:

```bash
docker run --help
docker build --help
docker image --help
```

This is useful for quickly checking:

- available options
- syntax
- examples

---

## 3. Common Docker Commands

Some of the most commonly used Docker commands are:

```bash
docker run
Docker container
Docker ps
Docker images
Docker build
```

### Important ones

- `docker run` → create and start a container from an image
- `docker ps` → list running containers
- `docker images` → list available images
- `docker build` → build an image from a Dockerfile

---

## 4. Build an Image

To build an image from a Dockerfile, use:

```bash
docker image build -t app:1.0 .
```

### Explanation

- `docker image build` → tells Docker to build an image
- `-t app:1.0` → gives the image a name and tag
- `.` → uses the current directory as the build context

The build context includes the files Docker will use while creating the image.

---

## 5. What Happens During Image Build

When you run a build, Docker performs several steps:

1. Sends the build context to the Docker daemon
2. Reads the Dockerfile instructions
3. Creates image layers
4. Produces the final image

These layers help make repeated builds faster and more efficient.

---

## 6. List Images

To view all images currently available on your system:

```bash
docker image ls
```

You can also use the shorter form:

```bash
docker images
```

This shows details such as:

- image name
- tag
- image ID
- size

---

## 7. Tag an Image

Tagging is used to assign a new name or version to an image.

```bash
docker image tag app:1.0 app:1.0-dev
```

### Example

- source image → `app:1.0`
- target image → `app:1.0-dev`

This is useful when you want to:

- version your image
- push it to a registry
- distinguish development and production images

---

## 8. Key Takeaways

For exam preparation, remember these core points:

- `docker version` verifies Docker installation
- `docker --help` is useful for command reference
- `docker build` creates an image from a Dockerfile
- `docker images` lists available images
- `docker image tag` assigns a new tag to an image

---

## 9. Quick Revision Summary

> Docker images are built from Dockerfiles, and basic commands like `docker build`, `docker images`, and `docker image tag` are enough for most CCAD/CKAD exam scenarios.
