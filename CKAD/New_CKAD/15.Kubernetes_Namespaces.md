# 🧩 Kubernetes Namespaces

This note explains what namespaces are in Kubernetes and how they help organize and isolate resources.

---

## 1. What is a Namespace?

A namespace provides a scope for Kubernetes resources.

It is similar to folders in a file system:

- a folder groups related files
- a namespace groups related Kubernetes resources

Think of it like this: if you have many applications in one cluster, you do not want all of them mixed together. Namespaces let you separate them into logical groups.

Example:

- `dev` namespace for development workloads
- `test` namespace for testing workloads
- `prod` namespace for production workloads

Resources that can belong to a namespace include:

- Pods
- Deployments
- Services
- ConfigMaps
- Secrets

---

## 2. Why Use Namespaces?

Namespaces are useful for:

- isolating resources
- separating environments such as dev, test, and prod
- controlling access for different users or teams
- organizing resources logically

They help keep the cluster structured and easier to manage.

---

## 3. Default Namespace

If a resource is created without an explicit namespace, it goes to the default namespace.

This means:

- the default namespace is used when no namespace is specified
- it is the fallback location for user-created resources

---

## 4. Types of Namespaces

Namespaces can be divided into two main categories:

### 1) Built-in Namespaces

These come with Kubernetes by default.

Common built-in namespaces include:

- `default` → default namespace for user resources
- `kube-system` → used by Kubernetes system components
- `kube-public` → used for publicly readable resources
- `kube-node-lease` → used for node lease information

### 2) User-defined Namespaces

These are created by users or administrators for organizing application resources.

Example:

- `dev`
- `test`
- `prod`

---

## 5. List Namespaces

To view all namespaces in the cluster:

```bash
kubectl get namespaces
```

You can also use the short form:

```bash
kubectl get ns
```

---

## 6. Create a Namespace

To create a new namespace:

```bash
kubectl create namespace dev
```

---

## 7. Use a Namespace

When creating a resource in a specific namespace, you can specify it with:

```bash
kubectl create namespace dev
kubectl run nginx --image=nginx -n dev
```

Or you can define it in the YAML manifest:

```yaml
metadata:
  namespace: dev
```

---

## 8. View Resources in a Namespace

To list resources in a particular namespace:

```bash
kubectl get pods -n dev
```

---

## 9. Delete a Namespace

To delete a namespace:

```bash
kubectl delete namespace dev
```

> Deleting a namespace also deletes the resources inside it.

---

## 10. Create Namespaces

To create a new namespace, use:

```bash
kubectl create namespace staging
```

You can also create another namespace such as production:

```bash
kubectl create namespace production
```

A shorter form is:

```bash
kubectl create ns production
```

---

## 11. List Namespaces

To list all namespaces in the cluster:

```bash
kubectl get namespaces
```

Short form:

```bash
kubectl get ns
```

This shows built-in namespaces as well as user-defined ones.

---

## 12. Describe a Namespace

To get high-level details about a namespace:

```bash
kubectl describe namespace staging
```

This shows details such as:

- labels
- annotations
- resource quota status
- limit ranges

---

## 13. View Namespace YAML

To view the YAML definition of a namespace:

```bash
kubectl get namespace staging -o yaml
```

This displays the namespace manifest in YAML format.

---

## 14. Assign a Namespace to a Resource

To create a resource inside a specific namespace, use the `-n` or `--namespace` flag.

Example:

```bash
kubectl run nginx --image=nginx -n staging
```

This creates a Pod named `nginx` inside the `staging` namespace.

You can create another Pod in the `production` namespace:

```bash
kubectl run nginx --image=nginx -n production
```

---

## 15. List Resources in a Specific Namespace

To list Pods in the current namespace:

```bash
kubectl get pods
```

To list Pods in a specific namespace:

```bash
kubectl get pods -n staging
```

You can also do the same for production:

```bash
kubectl get pods -n production
```

---

## 16. List Resources Across All Namespaces

To view resources from all namespaces:

```bash
kubectl get pods -A
```

The `-A` flag is shorthand for `--all-namespaces`.

---

## 17. Delete a Namespace

To delete a namespace:

```bash
kubectl delete namespace staging
```

When you delete a namespace, Kubernetes also deletes the resources inside that namespace.

---

## 18. Key Takeaway

Namespaces are used to group and isolate Kubernetes resources. They help with organization, access control, and separation of environments.

---

## 19. Quick Revision Summary

> Namespaces provide a scope for Kubernetes resources. They help isolate workloads, organize resources, and separate environments such as dev, test, and production.
